# Whisper_or_no_Whisper

# End-to-End Whisper Fine-Tuning and Optimization Project

Этот проект представляет собой полный пайплайн для дообучения (fine-tuning) и оптимизации моделей OpenAI Whisper для задачи автоматического распознавания речи (ASR).

## Возможности

-   **Гибкая настройка:** Все параметры (модели, пути, гиперпараметры обучения) вынесены в единый `config.yaml` файл.
-   **Полный цикл:** От подготовки данных до дообучения, оптимизации и финального тестирования.
-   **SOTA-модели:** Поддержка последних моделей Whisper, включая `large-v3`.
-   **Оптимизация для продакшена:** Автоматическая конвертация и квантизация модели в высокопроизводительный формат CTranslate2.
-   **Сравнительный анализ:** Наглядное сравнение производительности до и после оптимизации.

## Структура проекта

.

├── audio_files/            # Папка с вашими аудиофайлами (.wav)
├── my_dataset.json         # Файл с разметкой (путь к аудио | текст)
├── config.yaml             # Главный файл конфигурации
├── requirements.txt        # Зависимости проекта
├── run_pipeline.py         # Основной исполняемый скрипт
└── README.md               # Эта инструкция

## Установка и запуск

**1. Подготовка окружения:**

Рекомендуется использовать виртуальное окружение:

python -m venv venv
source venv/bin/activate  # Для Linux/macOS
# venv\Scripts\activate    # Для Windows

**2. Установка зависимостей:**

Установите все необходимые библиотеки из файла requirements.txt:

pip install -r requirements.txt
**3. Подготовка данных:**

-   Поместите ваши аудиофайлы (в формате .wav, моно, 16кГц) в папку audio_files/.
-   Создайте файл my_dataset.json со следующей структурой:
   
    [
        {"audio_path": "audio_files/sample_1.wav", "text": "первая транскрипция"},
        {"audio_path": "audio_files/sample_2.wav", "text": "вторая транскрипция"}
    ]
    
**4. Настройка конфигурации:**

Откройте файл config.yaml и настройте его под свою задачу. В первую очередь обратите внимание на секцию model, где вы можете выбрать базовую модель для дообучения.

**5. Запуск пайплайна:**

Запустите основной скрипт из терминала, указав путь к вашему файлу конфигурации:

python run_pipeline.py --config config.yaml
